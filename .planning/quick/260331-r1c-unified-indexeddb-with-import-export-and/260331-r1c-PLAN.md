  <action>
Create `app/lib/ibc-db.ts` â€” a client-side IndexedDB service for the single "ibc-tools" database (version 1).

**Stores:**
- `visualizador`: out-of-line keys (like current settings store). Keys: "theme", "playlist".
- `empaquetador`: keyPath "id", indexes: createdAt (non-unique), status (non-unique). Same structure as current packages store.

**Core helpers (re-export for consumers):**
- `openIbcDB(): Promise<IDBDatabase>` â€” opens "ibc-tools" v1, creates stores on upgrade.
- `getKey<T>(store: string, key: string): Promise<T | null>` â€” generic out-of-line key getter.
- `setKey(store: string, key: string, value: unknown): Promise<void>` â€” generic out-of-line key setter.
- `txStore(db: IDBDatabase, store: string, mode: IDBTransactionMode): IDBObjectStore` â€” transaction helper.

**Migration:**
- `migrateOldDatabases(): Promise<void>` â€” called once on first openIbcDB. Checks if "ibc-visualizador" or "ibc-empaquetador" exist (via indexedDB.open + check objectStoreNames), reads all data, writes into new DB stores, then calls `indexedDB.deleteDatabase()` for each old DB. Use a flag key `_migrated` in visualizador store to avoid re-running.

**Export/Import:**
- `exportToolData(tool: 'visualizador' | 'empaquetador' | 'all'): Promise<IbcToolsExport>` â€” reads specified store(s), returns `{ version: 1, tool, exportedAt: ISO string, data: { visualizador?: Record<string, unknown>, empaquetador?: unknown[] } }`.
- `importToolData(exportData: IbcToolsExport): Promise<void>` â€” validates version field, clears relevant store(s), writes imported data.
- `downloadExport(tool: 'visualizador' | 'empaquetador' | 'all'): Promise<void>` â€” calls exportToolData, creates JSON blob, triggers download as `ibc-tools-{tool}-{date}.ibctools`.
- `readImportFile(file: File): Promise<IbcToolsExport>` â€” reads File as text, JSON.parse, validates shape (version, tool, data fields). Throws descriptive error if invalid.

**Type:**
```typescript
export interface IbcToolsExport {
  version: 1;
  tool: 'visualizador' | 'empaquetador' | 'all';
  exportedAt: string;
  data: {
    visualizador?: Record<string, unknown>;
    empaquetador?: unknown[];
  };
}
```

Mark file with `'use client'` banner since it uses browser IndexedDB API. All functions should be async with try/catch and console.error for failures.
