    **1. Create `app/visualizador/lib/remote-types.ts`** — shared types for remote control protocol:

    ```typescript
    /** Command sent from mobile to desktop */
    export type RemoteCommand =
      | { type: 'NEXT_SLIDE' }
      | { type: 'PREV_SLIDE' }
      | { type: 'SET_PROJECTION_MODE'; mode: 'black' | 'clear' | 'logo' | 'slide' }
      | { type: 'SET_AUDIO_PLAYING'; playing: boolean };

    /** State snapshot pushed from desktop to room (lightweight for mobile display) */
    export interface RemoteState {
      activeHymnName: string;
      activeSlideLabel: string;
      activeSlideIndex: number;
      totalSlides: number;
      projectionMode: string;
      audioPlaying: boolean;
    }

    /** SSE event types */
    export type SSEEvent =
      | { event: 'state'; data: RemoteState }
      | { event: 'command'; data: RemoteCommand };
    ```

    **2. Create `app/api/visualizador/rooms/room-store.ts`** — in-memory room store:

    - Use a `Map<string, Room>` at module level.
    - `Room` shape: `{ id: string; pin: string; createdAt: number; lastActivity: number; state: RemoteState | null; desktopListeners: Set<WritableStreamDefaultWriter>; mobileListeners: Set<WritableStreamDefaultWriter> }`.
      - `desktopListeners` receives command events (mobile -> desktop).
      - `mobileListeners` receives state events (desktop -> mobile).
    - `generatePin()`: generate random 4-digit numeric PIN (1000-9999). Re-roll if already in use.
    - `createRoom()`: generate pin + crypto.randomUUID() id, add to map, return `{ pin, roomId }`.
    - `getRoom(pin)`: return room or null. Check expiry: if `Date.now() - lastActivity > 4 * 60 * 60 * 1000`, delete and return null.
    - `addListener(pin, type: 'desktop' | 'mobile', writer)`: add writer to appropriate Set, update lastActivity.
    - `removeListener(pin, type, writer)`: remove from Set.
    - `broadcastToDesktop(pin, command: RemoteCommand)`: encode as SSE `event: command\ndata: JSON\n\n`, write to all desktop listeners. Rate limit: track last 10 timestamps per room, reject if > 10/sec.
    - `broadcastToMobile(pin, state: RemoteState)`: encode as SSE `event: state\ndata: JSON\n\n`, write to all mobile listeners.
    - Run a cleanup sweep every 30 minutes via `setInterval` to remove expired rooms.

    **3. Create `app/api/visualizador/rooms/route.ts`** — POST handler:
    - Call `createRoom()`, return `{ pin, roomId }` with 201 status.
    - No auth required.

    **4. Create `app/api/visualizador/rooms/[pin]/stream/route.ts`** — GET SSE endpoint:
    - Accept query param `?role=desktop|mobile` (default: mobile).
    - Validate PIN exists via `getRoom(pin)`.
    - Return 404 if room not found.
    - Create a `TransformStream`, register the writable writer via `addListener`.
    - On stream close/abort, call `removeListener`.
    - If role=mobile and room has current state, send it immediately as first event.
    - Return `new Response(readable, { headers: { 'Content-Type': 'text/event-stream', 'Cache-Control': 'no-cache', 'Connection': 'keep-alive' } })`.
    - Send SSE keepalive comment (`: keepalive\n\n`) every 30 seconds via setInterval, clear on close.
    - Export `const dynamic = 'force-dynamic'`.

    **5. Create `app/api/visualizador/rooms/[pin]/command/route.ts`** — POST handler:
    - Parse JSON body, validate it matches RemoteCommand discriminated union (check `type` field is one of: NEXT_SLIDE, PREV_SLIDE, SET_PROJECTION_MODE, SET_AUDIO_PLAYING).
    - If type is SET_PROJECTION_MODE, validate `mode` is one of: black, clear, logo, slide.
    - Call `broadcastToDesktop(pin, command)`.
    - Return 200 `{ ok: true }` or 429 if rate limited or 404 if room not found.

    **6. Create `app/api/visualizador/rooms/[pin]/state/route.ts`** — POST handler:
    - Parse JSON body as RemoteState.
    - Validate required fields: activeHymnName (string), activeSlideLabel (string), activeSlideIndex (number), totalSlides (number), projectionMode (string), audioPlaying (boolean).
    - Store as `room.state = body` for late-joining mobiles.
    - Call `broadcastToMobile(pin, state)`.
    - Return 200 `{ ok: true }` or 404 if room not found.

    **2. Create `app/visualizador/hooks/useRemoteRoom.ts`:**

    Custom hook that manages the remote room lifecycle:

    ```typescript
    interface UseRemoteRoomOptions {
      onCommand: (cmd: RemoteCommand) => void;
    }
    interface UseRemoteRoomResult {
      pin: string | null;
      connected: boolean;
      error: string | null;
    }
    ```

    - On mount: POST to `/api/visualizador/rooms` to create room. Store `pin` in state.
    - After pin received: connect to `/api/visualizador/rooms/${pin}/stream?role=desktop` via EventSource.
    - Listen for `command` events, parse JSON, call `onCommand(cmd)`.
    - Track `connected` state based on EventSource readyState.
    - On EventSource error: set connected=false. EventSource auto-reconnects by default.
    - On unmount: close EventSource. (Room expires naturally after 4h.)
    - Expose a `pushState(state: RemoteState)` function that POSTs to `/api/visualizador/rooms/${pin}/state`. Debounce with 100ms trailing to avoid flooding on rapid slide changes.
    - Return `{ pin, connected, error, pushState }`.
