    **app/visualizador/components/AudioBar.tsx** -- `'use client'` component. This is the bottom bar (72px height, full width per D-02 and UI-SPEC). The `<audio>` element lives here and persists across hymn/slide changes (per Research pitfall 5 -- avoid remount).

    Props:
    ```typescript
    interface AudioBarProps {
      hymnAudio: HymnAudioFiles | null; // audio files for current hymn, null if no hymn selected
      hymnId: string | null; // ID of current hymn (to detect hymn changes per D-20)
      activeTrackField: string | null; // which track is selected
      playing: boolean;
      onTrackChange: (trackField: string) => void;
      onPlayingChange: (playing: boolean) => void;
    }
    ```

    Internal state:
    - `audioRef = useRef<HTMLAudioElement>(null)` -- persistent audio element
    - `progress: number` (0-100), `currentTime: number`, `duration: number` via useState
    - `prevHymnId = useRef<string | null>(null)` -- track hymn changes

    Audio element behavior:
    - Render `<audio ref={audioRef} preload="metadata" />` (no src initially).
    - When `activeTrackField` or `hymnId` changes, compute new src: if `hymnAudio && activeTrackField && hymnAudio[activeTrackField as keyof HymnAudioFiles]`, set `audioRef.current.src = /api/hymns/audio/${fileInfo.id}`. Otherwise clear src.
    - When `hymnId` changes from previous value (check via prevHymnId ref): pause audio, reset progress/time to 0, update prevHymnId. Then set default track: find first available track preferring `track_only` first (per D-18), then `midi_file`, then voice parts. Call `onTrackChange(defaultField)`.
    - When `playing` prop changes: if true, call `audioRef.current.play()`. If false, `audioRef.current.pause()`.
    - Audio events: `onPlay -> onPlayingChange(true)`, `onPause -> onPlayingChange(false)`, `onEnded -> onPlayingChange(false); reset progress`, `onTimeUpdate -> update progress and currentTime`, `onLoadedMetadata -> update duration`.

    Layout (72px height, per UI-SPEC):
    ```
    [Play/Pause btn] | [Track selector dropdown] | [Seek bar (Slider)] | [Time display]
    ```
    - Horizontal flex, items-center, px-4, gap-4, border-t border-border, h-[72px], bg-card.
    - **Play/Pause**: shadcn `Button` variant="ghost" size="icon". Icon: `Play` or `Pause` from Lucide. `aria-label` "Reproducir" or "Pausar". Wrapped in `Tooltip` with label + "(P)" keyboard hint.
    - **Track selector**: shadcn `DropdownMenu` with `DropdownMenuTrigger` as a Button showing current track label (e.g., "Pista completa"). `DropdownMenuContent` lists only available tracks (iterate `hymnAudio` keys, skip null values, skip `midi_file` for NON_PLAYABLE). Each `DropdownMenuItem` shows `AUDIO_LABELS[field]`. On select, call `onTrackChange(field)`. Label above: "Pista de audio" in `text-xs text-muted-foreground` (12px label per UI-SPEC).
    - **Seek bar**: shadcn `Slider` component. `min={0}`, `max={100}`, `step={0.1}`, `value={[progress]}`. On `onValueChange`, seek audio: `audioRef.current.currentTime = (value[0] / 100) * duration`. Height 44px touch target (per UI-SPEC spacing exception). `aria-label="Progreso de audio"`.
    - **Time display**: `text-xs tabular-nums text-muted-foreground` showing `formatTime(currentTime) / formatTime(duration)`. Use same `formatTime` helper as HymnDetailModal: `m:ss` format.

    Empty/hidden state: If `hymnAudio` is null or no hymn selected, render the bar with reduced opacity (opacity-50) and disabled controls. Show "Sin pistas de audio disponibles para este himno." text if hymn selected but has no audio.

    Audio entrance animation per UI-SPEC: Wrap the entire bar in `<motion.div>` with `initial={{ y: 72 }}`, `animate={{ y: 0 }}`, `transition={{ duration: 0.2, ease: 'easeOut' }}` for "slide up from bottom" when first hymn is selected. Use Framer Motion.
