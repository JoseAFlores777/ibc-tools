### Projection Window (fullscreen output)

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Verse label | 24px | 400 (regular) | 1.2 | "ESTROFA I", "CORO" — subtle, above lyrics |
| Lyrics | auto-sized (16px-120px) | 400 (regular) | 1.4 | Main projected text, binary-search sized to fill screen |
| Title slide: hymn name | auto-sized (32px-80px) | 600 (semibold) | 1.2 | Hymn name on cover slide |
| Title slide: reference | 24px | 400 (regular) | 1.4 | Bible reference below hymn name |

Font auto-sizing algorithm: binary search between 16px min and 120px max using canvas measureText. Container area = viewport minus 80px padding per side. Operator manual adjustment via Ctrl+Plus/Ctrl+Minus applies a size offset (-20 to +20) to the auto-calculated base.

## Layout Contract

### Control Panel (3-column + bottom bar)

```
+---------------------------------------------+
| [Left Column]  [Center Column]  [Right Col]  |
| 280px fixed    1fr flexible     320px fixed   |
| Playlist +     Slide thumbnail  Live preview  |
| search         grid             + projection  |
|                                 controls      |
+---------------------------------------------+
| [Audio Bar — full width, 72px height]         |
| play/pause | track selector | seek bar | time |
+---------------------------------------------+
```

| Region | Width | Height | Scroll |
|--------|-------|--------|--------|
| Left column (playlist) | 280px fixed | h-screen minus audio bar (72px) | ScrollArea vertical |
| Center column (slide grid) | flex-1 (fill remaining) | h-screen minus audio bar | ScrollArea vertical |
| Right column (preview + controls) | 320px fixed | h-screen minus audio bar | No scroll (preview scales to fit) |
| Audio bar | 100% | 72px fixed | No scroll |

Column separators: 1px `border-border` (dark theme resolved) vertical lines between columns.

### Projection Window (fullscreen)

- Viewport fills entire screen via Fullscreen API
- No browser chrome, no controls, no scrollbars
- Slide content centered vertically and horizontally
- 80px safe-area padding on all sides
- Background fills entire viewport (image or solid/gradient)
- Crossfade transition: 400ms via Framer Motion AnimatePresence

## Interaction Contract

### Keyboard Shortcuts

| Key | Action | Context | preventDefault |
|-----|--------|---------|----------------|
| ArrowRight | Next slide | Always | Yes |
| ArrowLeft | Previous slide | Always | Yes |
| Space | Next slide | Always | Yes |
| Escape | Close projection / exit fullscreen | When projection open | No (browser handles) |
| B | Black screen toggle | Always | Yes |
| L | Logo toggle | Always | Yes |
| C | Clear text toggle | Always | Yes |
| Ctrl+Plus | Increase font size | Always | Yes |
| Ctrl+Minus | Decrease font size | Always | Yes |
| P | Play/pause audio | Always | Yes |

### Drag-and-Drop

- Library: @dnd-kit/sortable with verticalListSortingStrategy
- Drag handle: grip icon (GripVertical from Lucide) on left side of playlist item
- Visual feedback during drag: item lifts with subtle shadow, placeholder shows drop position
- Touch support: included via @dnd-kit

### Slide Navigation Flow

1. Click hymn in playlist -> loads slides in center grid, first slide auto-projects
2. Click thumbnail in grid -> projects that slide
3. ArrowRight on last slide of hymn -> auto-advances to first slide of next hymn in playlist (D-07)
4. ArrowLeft on first slide of hymn -> goes to last slide of previous hymn in playlist

### Projection Window Lifecycle

1. Click "Proyectar" -> `window.open('/visualizador/proyeccion', 'ibc-projection', 'popup=true')` (synchronous in click handler)
2. Projection page loads -> shows "Haga clic para pantalla completa" overlay
3. User clicks overlay -> `document.documentElement.requestFullscreen()`
4. Projection sends `PING` via BroadcastChannel -> control panel responds with full state
5. Control panel polls `projWindow.closed` every 1000ms to detect close

### Audio Behavior

- Audio `<audio>` element lives in AudioBar (persists across hymn/slide changes)
- Switching slides within same hymn: audio continues (D-20)
- Switching to different hymn: audio stops, new hymn's default track loads (D-18: `track_only` preferred)
- Track selector shows only available tracks for current hymn (D-18)
- Audio and slides are independent — no sync (D-16)

## Transition and Animation Contract

| Trigger | Animation | Duration | Easing | Library |
|---------|-----------|----------|--------|---------|
| Slide change (projection) | Crossfade (opacity 0->1, old 1->0) | 400ms | ease-in-out | Framer Motion AnimatePresence |
| Slide change (preview) | Instant (no animation) | 0ms | N/A | None |
| Playlist item drag | Translate Y with shadow lift | Drag duration | spring(stiffness:200) | @dnd-kit CSS transform |
| Playlist item add | Fade in + slide down | 200ms | ease-out | Framer Motion |
| Playlist item remove | Fade out + slide up | 150ms | ease-in | Framer Motion |
| Projection mode change (Negro/Limpiar/Logo) | Fade to black then fade to new content | 300ms | ease-in-out | Framer Motion |
| Audio bar appear (first hymn selected) | Slide up from bottom | 200ms | ease-out | Framer Motion |

Reduced motion: when `prefers-reduced-motion` is detected, all transitions become instant (duration 0ms). Framer Motion's `useReducedMotion` hook handles this.

## Responsive Strategy

This is primarily a desktop tool (D-03: full-height 3-column layout). Minimum supported viewport: 1024px wide.

| Viewport | Layout |
|----------|--------|
| >= 1280px | 3-column layout as designed (280px + 1fr + 320px) |
| 1024px - 1279px | 3-column with compressed widths (240px + 1fr + 280px) |
| < 1024px | Show message: "El visualizador requiere una pantalla de al menos 1024px de ancho. Use una computadora de escritorio o portatil." |

The projection window is always fullscreen and has no responsive breakpoints — it fills whatever screen it is displayed on.
