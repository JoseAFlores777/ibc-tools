# Phase 2 — UI Design Contract

> Visual and interaction contract for PDF document rendering. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase scope:** This phase produces server-side PDF components rendered via `@react-pdf/renderer`. There is no browser UI, no interactive elements, and no web pages. The "UI" is the printed PDF document layout. All values below are in PDF points (1pt = 1/72 inch) and apply to `@react-pdf/renderer` `StyleSheet.create()` styles, NOT Tailwind/CSS.

---

| Property | Value |

## Page Dimensions

All layouts use LETTER paper (612 x 792 pt / 8.5 x 11 in).

| Layout | Page Padding | Usable Width | Usable Height | Column Width |
|--------|-------------|--------------|---------------|-------------|
| 1-per-page | 40pt all sides | 532pt | 712pt | 532pt (full) |
| 2-per-page | 20pt all sides | 572pt | 752pt | 277pt per column |

**2-per-page divider:** 1pt line with 8pt margin on each side = 17pt total. Each column = (572 - 17) / 2 = 277.5pt.

**Source:** CONTEXT.md D-04, D-05, RESEARCH.md Page Dimension Reference

## Spacing Scale

Declared values for PDF layout spacing (in points):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Inline gaps (e.g., between hymn number and title text) |
| sm | 8pt | Column divider margins, compact element separation |
| md | 12pt | Verse block bottom margin (separation between verses) |
| lg | 20pt | Page padding (2-per-page layout), body section top padding |
| xl | 40pt | Page padding (1-per-page layout), horizontal section padding |

Exceptions: Footer height is fixed at 100pt for decorated style (matches existing HymnPagePdf). Header height is content-driven (auto).

## Overflow Handling (2-Per-Page)

When hymn lyrics exceed the available column height:

1. **Primary strategy:** Use the reduced font sizes specified in the 2-per-page typography tables (8-9pt). This handles most hymns (up to ~6 verses).
2. **Overflow strategy:** If content still exceeds column height, allow react-pdf's natural page break. The hymn continues on the next page. The partner column on the overflow page remains empty.
3. **No truncation.** Lyrics are never cut off.

**Source:** RESEARCH.md Open Question 2, Claude's Discretion area.

**Empty states:** Not applicable -- this phase generates PDFs from existing hymn data. Empty/missing data handling:
- Missing hymn number: Display "Himno" without number
- Missing hymnal: Omit hymnal name line
- Missing bible text/reference: Omit biblical verse section in header
- Missing authors: Omit author lines in footer
- Empty lyrics: Render page with header/footer only (no body content)

## Design Tokens (react-pdf StyleSheet constants)

These named constants must be used in all PDF components instead of magic numbers:

```typescript
// Page dimensions
const LETTER_WIDTH = 612;   // pt
const LETTER_HEIGHT = 792;  // pt

// Margins
const MARGIN_1UP = 40;      // pt -- 1-per-page padding
const MARGIN_2UP = 20;      // pt -- 2-per-page padding

// Divider
const DIVIDER_WIDTH = 1;    // pt -- vertical line
const DIVIDER_MARGIN = 8;   // pt -- gap each side of divider
const DIVIDER_TOTAL = 17;   // pt -- 1 + 8 + 8

// Column width (2-per-page)
const COLUMN_WIDTH_2UP = Math.floor((LETTER_WIDTH - 2 * MARGIN_2UP - DIVIDER_TOTAL) / 2);

// Footer (decorated only)
const FOOTER_HEIGHT = 100;  // pt
const FOOTER_BORDER_TOP = 9; // pt -- gold border
const HEADER_BORDER_BOTTOM = 7; // pt -- gold border (1-up)
const HEADER_BORDER_BOTTOM_2UP = 5; // pt -- reduced for 2-up

// Colors
const COLORS = {
  headerBg: '#393572',
  headerOverlay: 'rgba(53, 73, 115, 0.85)',
  goldAccent: '#9e7f19',
  goldHighlight: '#eaba1c',
  pageBg: '#f7f7f7',
  bodyText: '#444444',
  lightText: '#c2c2c4',
  headerText: '#ffffff',
  footerBg: '#2E4067',
  divider: '#cccccc',
  plainSubtitle: '#666666',
};
```
