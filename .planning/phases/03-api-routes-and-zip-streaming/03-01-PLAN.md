**Step 2: Create `app/lib/zip/zip.schema.ts`** (per D-01 archiver, Zod for validation)

Export a Zod schema `packageRequestSchema` and inferred type `PackageRequest`:

```typescript
import { z } from 'zod';

export const packageRequestSchema = z.object({
  hymns: z
    .array(
      z.object({
        id: z.string().uuid(),
        audioFiles: z
          .array(z.enum([
            'track_only',
            'midi_file',
            'soprano_voice',
            'alto_voice',
            'tenor_voice',
            'bass_voice',
          ]))
          .optional()
          .default([]),
      }),
    )
    .min(1, 'Al menos un himno es requerido')
    .max(50, 'Maximo 50 himnos por paquete'),
  layout: z.enum(['one-per-page', 'two-per-page']),
  style: z.enum(['decorated', 'plain']),
});

export type PackageRequest = z.infer<typeof packageRequestSchema>;
```

**Step 3: Create `app/lib/zip/generate-hymn-zip.ts`**

This is the core ZIP assembly logic. Must export:

1. `createStreamingZip()` -- creates archiver instance piped to PassThrough, converts to Web ReadableStream via `Readable.toWeb()`. Returns `{ archive, webStream }`. Per D-01 and D-02. Use `zlib: { level: 5 }` for balanced speed/compression.

2. `hymnFolderName(hymnNumber: number | null, name: string, id: string): string` -- per D-04: `{hymn_number} - {hymn_name}` sanitized for filesystem (replace `/\:*?"<>|` with `_`). If hymn_number is null, use `himno-{id}`.

3. `assembleHymnPackage(archive: archiver.Archiver, request: PackageRequest): Promise<{ successCount: number; errorCount: number }>` -- the main assembly function that:
   - Iterates over `request.hymns` sequentially
   - For each hymn: calls `fetchHymnForPdf(hymn.id)` to get full data
   - Generates individual PDF via `renderHymnPdf({ hymns: [hymnData], layout: request.layout, style: request.style })` and appends to archive as `{folderName}/{folderName}.pdf` (per D-06)
   - For each audio field in `hymn.audioFiles`: if the field exists in `hymnData.audioFiles` (not null), downloads from Directus via `fetch(getAssetUrl(fileInfo.id))`, converts response body to Node stream via `Readable.fromWeb(response.body as any)`, appends to archive as `{folderName}/{filename_download}` (per D-05). If `filename_download` is falsy, use `{field_name}.{extension from type}` as fallback.
   - Per D-07: if PDF render fails, catch error, add to errors array, do NOT fail
   - Per D-08: if audio download fails (non-ok response or network error), catch error, add to errors array, do NOT fail
   - If errors array is non-empty for a hymn, append `{folderName}/ERROR.txt` with joined error messages
   - After individual hymns: generate combined PDF via `renderHymnPdf({ hymns: allSuccessfulHymns, layout, style })` and append at ZIP root as `himnos.pdf` (per D-06). If combined PDF fails, log and skip.
   - After all entries: call `archive.finalize()`
   - Return `{ successCount, errorCount }` for the caller to check D-09 (all fail = 500)
   - Track `successCount` (hymns with at least one successful entry) and `errorCount`

**Step 1: Create `app/api/hymns/search/route.ts`** (per D-12, D-13)

Follow the exact pattern from `app/api/events/route.ts`:

```typescript
import { NextResponse } from 'next/server';
import { searchHymns } from '@/app/lib/directus/services/hymns';

export const dynamic = 'force-dynamic';

export async function GET(request: Request) {
  try {
    const { searchParams } = new URL(request.url);
    const q = searchParams.get('q') || undefined;
    const hymnal = searchParams.get('hymnal') || undefined;
    const category = searchParams.get('category');
    const limit = Number(searchParams.get('limit')) || 25;
    const offset = Number(searchParams.get('offset')) || 0;

    const results = await searchHymns({
      query: q,
      hymnalId: hymnal,
      categoryId: category ? Number(category) : undefined,
      limit,
      offset,
    });

    return NextResponse.json(
      { ok: true, data: results },
      {
        headers: {
          'Cache-Control': 's-maxage=60, stale-while-revalidate=30',
        },
      },
    );
  } catch (error: any) {
    console.error('GET /api/hymns/search error:', error?.message || error);
    return NextResponse.json(
      { ok: false, error: 'Error al buscar himnos' },
      { status: 500 },
    );
  }
}
```

Key details per D-12:
- Query params: `q` (text search), `hymnal` (hymnal UUID), `category` (category ID as number), `limit` (default 25), `offset` (default 0)
- Maps `q` -> `query`, `hymnal` -> `hymnalId`, `category` -> `categoryId` (parsed as number)
- Returns `{ ok: true, data: HymnSearchResult[] }` on success
- Returns `{ ok: false, error: 'Error al buscar himnos' }` with status 500 on failure
- Cache-Control: `s-maxage=60, stale-while-revalidate=30` (search results cacheable for 1 min)
